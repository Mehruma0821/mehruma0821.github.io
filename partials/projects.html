LRS2 Lip-Reading Database & Web Portal

Solo Developer & Researcher · Supervised by Dr. Daqing Chen, London South Bank University

A full-stack data engineering project that turns the raw LRS2 lip-reading video corpus — 144,483 video clips with no structured way to search or analyse them — into a searchable, publicly accessible research database supporting lip-reading and homophone-disambiguation research.

Database Design

Designed an 18-table relational schema (ERD) modelling videos, words, homophones, visemes, phonemes and speakers. Captures viseme groups (lip shapes shared by multiple phonemes, e.g. B/M/P) and homophone pairs — the core ambiguity problem in lip-reading AI.

Data Pipeline

Python extraction pipeline processed all 144,483 raw transcript files with zero errors, verified against an independent 48,164-sentence reference set. Tagged 37,472 videos containing homophones and migrated 2.4M+ rows (464MB) into SQL Server.

Web Application

Flask portal with multi-filter search (keyword, duration, dataset split, homophone presence, size), paginated results, and detail pages with word-by-word homophone highlighting. On-demand FFmpeg transcoding (MPEG-4 → H.264) with range-request video streaming.

Cloud Migration & Deployment

Migrated the database from SQL Server to PostgreSQL (Neon) and deployed to Render with environment-based config. Two-tier architecture: a full local version with video playback, and a lightweight public version — a deliberate scope decision around hosting 49.6GB of video on free infrastructure.
